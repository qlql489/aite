这一章聚焦 Aite 的核心使用流程，也就是“项目管理 + Claude Code 聊天协同”。

如果说“快速开始”解决的是能不能跑起来，那么这一章解决的是怎么把 Aite 真正用顺手。

## 这一章覆盖什么

### 项目管理

- 从 Claude Code 历史记录导入项目
- 手动导入新的本地文件夹
- 项目搜索、排序与删除

### 聊天与会话

- 历史对话加载
- 活跃 Session 管理
- 会话删除与恢复

### 运行控制

- 工具调用显示
- 权限审批
- 权限模式
- 思考强度
- 模型切换
- 思考内容展示

### 工作区协同

- Git 分支与工作区
- IDE 连接
- 文件树浏览与文件编辑
- 项目级 Command、Skill、MCP 管理

## Aite 的主工作流

大多数情况下，你会按下面这条路径使用 Aite：

1. 导入项目
2. 选择或恢复一个会话
3. 查看历史消息
4. 继续发送消息
5. 在聊天流里处理工具调用和权限审批
6. 结合 Git、IDE、文件树和扩展能力继续工作

## 和纯 CLI 工作流的区别

和直接使用 Claude Code CLI 相比，Aite 主要提供了这几层增强：

- 项目列表化管理
- 历史会话可视化
- 工具调用结构化展示
- 权限审批内嵌到聊天流
- 模型、权限模式、思考强度的界面化控制
- 文件树、IDE、Git 与扩展能力整合到同一工作区
