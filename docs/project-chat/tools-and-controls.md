# 聊天工具与运行控制

这一页聚焦聊天输入区和消息区里的“控制面板能力”，也就是你在对话过程中最常操作的那些开关和状态。

## 工具调用如何展示

Aite 会把 Claude Code 的工具调用拆成独立块展示，而不是把所有中间过程都混进自然语言文本里。

这意味着你能更容易看清：

- 调用了什么工具
- 工具输入是什么
- 当前是运行中、成功、失败还是等待审批
- 返回结果是什么

对于文件类工具，Aite 还会尽量把路径做相对化展示，减少长绝对路径带来的阅读负担。

## Diff 与结构化结果

当工具涉及文件写入、编辑或补丁结果时，Aite 会提供更适合阅读的结构化展示方式，例如 diff 视图。

这类展示的意义是：

- 更快看出模型到底改了什么
- 区分“看起来说了很多”与“实际改动很少”
- 降低审核成本

## 子任务与子代理信息

如果某段执行里包含子任务、子代理或工具链式过程，Aite 会尽量把它们作为独立消息块呈现，而不是全部压平。

这有助于判断：

- 当前工作是不是在主链路上
- 某个子步骤是否真的完成
- 中途失败是否影响最终结果

## 权限审批

权限审批是 Aite 聊天体验里非常关键的一部分。

当 Claude Code 请求额外操作权限时，审批入口会直接出现在聊天流中，而不是跳到外部终端。

常见审批动作包括：

- `允许`
- `始终允许`
- `拒绝`

### `允许`

只批准当前这一次请求。

### `始终允许`

除了批准当前请求，还会把这类规则记下来，减少后续重复确认。

### `拒绝`

拒绝这次操作；在某些场景下还可以附带拒绝原因或建议。

## 权限模式

Aite 允许你在会话级别切换权限模式。

权限模式的作用，可以简单理解为“Claude Code 在当前会话里默认能走多远，需要多频繁地来问你确认”。

不同任务适合不同模式：

- 谨慎审查型任务：适合更保守的模式
- 快速迭代型任务：适合减少频繁确认的模式

建议按任务风险来选，不要长期一刀切。

## 思考强度

Aite 支持切换思考强度，常见层级包括：

- 关闭思考
- 浅思考
- 中思考
- 深思考

它更像是在控制模型在响应前愿意投入多少推理预算。

一般可以这样理解：

- 简单改动、机械任务：浅思考或关闭
- 普通编码任务：中思考
- 复杂分析、重构、疑难排查：深思考

## 模型切换

当前聊天输入区支持直接选择“供应商 - 模型”组合。

这意味着你可以在同一个界面里快速切换：

- 使用哪个供应商
- 使用该供应商下的哪个模型

这比单独分别切供应商和模型更直观，也更接近日常决策方式。

## 思考内容显示

如果当前会话和模型支持，Aite 会把思考内容显示出来，方便你判断：

- 模型是不是理解偏了
- 某个问题为什么卡住
- 是否需要换模型、提权或补充上下文

不是每次都要看，但在排查复杂问题时非常有用。

## 运行状态

除了消息本身，Aite 还会展示执行状态相关信息，例如：

- 运行中
- 已完成
- 失败
- 等待审批

对于复杂任务，这些状态比单纯看最后一条文字更重要，因为你能很快判断“到底是没做完，还是做完了但结果不理想”。
