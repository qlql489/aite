# 项目导入与管理

这一页说明 Aite 如何组织本地项目，以及项目列表中的常见操作。

## 项目从哪里来

Aite 当前支持两种主要导入方式：

### 1. 从 Claude Code 历史导入

这是最有代表性的方式。

如果你已经在某个目录里使用过 Claude Code，Aite 可以扫描这些历史项目，并把它们导入到项目列表中。

导入弹窗里通常会展示：

- 项目名称
- 项目路径
- 已存在的会话数量
- 最近活动时间
- 对话文件总大小
- 对应的 Claude CLI 版本

这类导入特别适合从 CLI 工作流切换到桌面工作流的用户。

### 2. 导入新的本地文件夹

如果某个项目还没有 Claude Code 历史记录，也可以直接手动导入本地目录。

这种方式适合：

- 新项目刚开始接入 Aite
- 历史扫描没覆盖到的目录
- 想把常用工作目录统一放到项目列表中管理

## 项目列表怎么用

项目列表不是单纯的文件夹展示，它更接近“Claude 工作入口”。

每个项目下面会继续挂载它的会话列表，所以你通常不只是打开一个目录，而是在进入某个项目的上下文。

## 项目搜索

项目列表支持搜索，主要用于按下面两类信息筛选：

- 项目名称
- 项目路径

当本地项目比较多时，这个搜索会非常有用，尤其是多个同名仓库或不同分支工作副本并存的时候。

## 项目排序

项目页会结合项目和会话的活动情况组织顺序，核心目标是让最近常用的内容更容易出现在前面。

实际使用中，你可以把它理解成：

- 最近活跃的项目更容易被优先看到
- 有正在进行中的会话时，项目更容易保持在工作视线内

## 项目内会话展开

一个项目可以包含多个历史会话。

通常你会在项目列表中先展开项目，再选择具体会话，而不是先进入项目后再从空白状态新建。

这样做的好处是：

- 能快速找到上下文连续的旧对话
- 能区分不同任务分支
- 能同时保留多个项目的工作现场

## 删除项目

删除项目的含义是“从 Aite 项目列表中移除这个项目”。

需要注意：

- 项目会从界面列表中移除
- 与该项目关联的会话条目也会从当前列表中移除
- 这不等于删除你本地磁盘上的项目代码目录

因此，删除项目更像是清理工作台，而不是删仓库。

## 适合的管理习惯

比较推荐的做法是：

- 把长期维护的仓库常驻在项目列表
- 把临时项目按需导入
- 完成阶段性工作后清理不再需要的项目展示项

这样项目区会更干净，也更接近日常真实工作流。
