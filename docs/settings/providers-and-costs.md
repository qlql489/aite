# 供应商与费用统计

这一页说明设置里的“供应商”和“统计”两部分。

它们共同决定两件事：

- Claude Code 走哪家模型能力
- 你大概花了多少 Token 和费用

## 供应商管理

Aite 支持在设置页里统一管理模型供应商。

从当前实现看，供应商配置页支持：

- 新建供应商
- 编辑供应商
- 复制供应商
- 删除供应商
- 维护模型列表
- 维护额外环境变量

## 一个供应商通常包含什么

一条供应商配置通常包括：

- 名称
- 首页地址
- Base URL
- 协议类型
- 鉴权类型
- API Key
- 模型列表
- 主模型
- 额外环境变量

这意味着不只是官方供应商能接，兼容协议的第三方供应商也可以纳入统一管理。

## 协议与兼容格式

当前实现里，供应商支持不同协议类型与上游格式。

简单理解就是：

- 有些供应商更接近 Anthropic 协议
- 有些供应商走 OpenAI 兼容协议

Aite 会在运行时做必要桥接，尽量让你在界面层统一管理，而不是手工拼环境变量。

## 默认供应商与系统继承

供应商页里有一个很关键的开关：继承系统配置。

### 开启继承系统配置

表示优先使用系统已有环境，而不是由 Aite 注入自定义供应商配置。

这适合：

- 你已经有成熟的 CLI / shell 配置
- 你还在和其他工具共享同一套供应商环境

### 关闭继承系统配置

表示由 Aite 接管默认供应商选择和相关环境注入。

这适合：

- 希望在 GUI 里统一管理供应商
- 不想手工维护复杂环境变量
- 希望聊天里直接按“供应商 - 模型”切换

## 模型列表与主模型

每个供应商可以维护自己的模型列表，并指定一个主模型。

这个主模型会影响：

- 默认进入会话时的选择
- 聊天区模型切换时的可选项来源

因为聊天输入区现在已经改成“供应商 - 模型”组合选择，所以供应商配置的质量会直接影响日常使用体验。

## 费用统计

设置页里的统计模块用于从全局视角看使用成本和 Token 消耗。

当前实现已经支持按多个维度查看统计结果，例如：

- 概览
- 按模型
- 按会话
- 时间线

## 项目范围与时间范围

统计模块支持：

- 所有项目视角
- 单项目视角
- 最近 7 天
- 最近 30 天
- 全部时间

这让你既可以看整体花费，也可以定位某个项目或某段时间的成本变化。

## 统计适合用来做什么

它最适合解决下面这些问题：

- 最近为什么费用上涨了
- 哪个模型最贵
- 哪个项目最消耗 Token
- 哪个会话成本异常高

## 一条实际建议

如果你接了多个供应商，建议定期同时看两件事：

1. 当前默认供应商是否还是最合适的
2. 成本是否和你的实际收益匹配

这样“模型选择”就不只是凭感觉，而是可以结合使用数据来调。
