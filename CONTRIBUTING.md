# Contributing

感谢你关注 Aite。

这份文档主要用于统一提交流程、问题反馈方式与基础开发约定。

## 反馈问题

提交问题时，建议尽量附带这些信息：

- 使用平台与系统版本，例如 `macOS 14`、`Windows 11`
- Aite 版本号
- Claude Code CLI 版本
- 是否首次启动、是否刚执行过更新
- 复现步骤
- 相关截图或日志片段

## 提交变更

如果你准备提交代码，请尽量遵循下面的原则：

- 变更尽量聚焦单一问题，避免把无关修改混在同一个提交里
- 优先保证用户可感知行为稳定，再考虑内部重构
- 涉及 UI 变更时，补充说明影响范围
- 涉及 Windows、macOS 差异逻辑时，注明平台行为差异
- 更新文案、功能入口、安装方式时，同步更新 README 或相关文档

## 提交信息建议

推荐使用语义化前缀：

- `feat:` 新功能
- `fix:` 缺陷修复
- `docs:` 文档更新
- `refactor:` 重构
- `chore:` 杂项维护

## 合并前检查

在发起合并前，建议至少确认：

- 能正常安装依赖并完成本地构建
- 核心聊天流程可以正常创建或恢复会话
- 权限审批、文件引用、扩展页入口没有明显回归
- 文档、配置项、版本信息已同步

## 行为准则

请保持清晰、友好、可协作的沟通方式，优先围绕问题本身推进讨论。
