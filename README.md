## Aite 是什么

Aite 是一个围绕 Claude Code 使用体验打造的桌面客户端。

它把日常会遇到的这些高频动作整合到了一起：

- 选择和管理本地项目
- 创建、恢复、切换和搜索会话
- 在聊天中查看思考、工具调用、任务和子代理过程
- 对编辑、命令和计划执行进行权限审批
- 在输入框里直接引用文件、目录、图片和附件
- 管理供应商、MCP、skills、commands 和更新设置

如果你已经在用 Claude Code，Aite 更像是一个更适合长期使用的桌面工作台。

## 为什么好用

### 多会话更清楚

- 多个会话独立运行，互不干扰
- 支持会话刷新、恢复历史会话、`--resume` 延续上下文
- 会话标题自动生成，侧边栏切换更快
- 支持草稿会话复用、删除对话、欢迎页快速开始
- 会话搜索、项目搜索、已导入项目搜索都可用

### 权限控制更顺手

- 支持计划模式、编辑模式、只读模式、绕过权限模式
- 权限请求可以在聊天流里直接审批
- 工具调用卡片旁可直接批准或拒绝
- 支持会话内“始终允许”
- 权限模式切换和历史会话重连后的同步都已打通

### 聊天体验更完整

- 默认启用流式响应，消息显示更稳定
- 支持 thinking、tool use、tool result、图片、任务、子代理等多种消息类型
- 工具调用结果支持结构化展示和展开查看
- Edit / Write 支持 diff 视图
- Read 支持行号显示
- AskUserQuestion 有独立交互卡片
- compact 后聊天历史、父节点链路和回退流程都做了恢复和修复

### 文件引用更自然

- 输入框里支持 `@` 文件引用
- 支持目录引用和目录附件
- 文件搜索会和输入状态联动
- 文件 chip 做了更清晰的展示优化
- 图片和通用附件都能直接带入对话

### 项目工作区更实用

- 支持本地项目导入与管理
- 支持工作区目录树查看
- 支持内联文件编辑器
- 集成 Git 状态、分支菜单和工作区变更信息
- 支持 worktree 模式，并在会话结束后自动清理

### 扩展能力更集中

- MCP Server 支持状态检测、开关控制、缓存和延迟刷新优化
- skills / commands 可统一管理
- 扩展页采用分 Tab 组织
- 支持 skill 导入和 skill 文件树查看
- 斜杠命令支持即时执行和分组展示

### 统计与设置更完整

- 提供统计页，支持按项目查看
- 支持异步加载、日期筛选和上下文占比计算
- 可查看最后一条 assistant 的 token 使用情况
- 设置页支持 Claude CLI 参数管理
- 支持自动更新流程
- 支持 macOS 双架构打包和公开发布流程

## 安装说明

- macOS 11.2 及以上、Windows 10 及以上可用
- 首次启动会检查 Claude Code CLI 及所需运行环境
- 如果本机缺少 Node.js / npm，Aite 会根据安装流程提示补齐
- 更新版本后，建议重启应用一次，确保桌面端与 CLI 状态同步

## 常见问题

### 需要提前安装 Claude Code CLI 吗

不一定。

如果本机尚未安装，Aite 会在首次使用相关能力时进行检测，并在需要时引导安装。

### 为什么首次启动会检测 Node.js / npm

因为 Claude Code CLI 依赖对应运行环境。如果本机缺失，Aite 会先提示补齐，再继续后续安装流程。

### 能直接拿来管理多个项目吗

可以。Aite 的核心就是把多项目、多会话、多种权限流转放进同一个桌面工作台里。

### 更新后需要重新配置吗

一般不需要。大部分项目、会话和设置会继续保留；如果更新内容涉及安装链路、供应商配置或扩展能力，建议重新打开应用确认一次状态。

## 快速开始

1. 打开 Aite
2. 导入一个本地项目
3. 新建会话或恢复历史会话
4. 在输入框中直接开始提问，按需引用文件、目录或图片
5. 在需要时审批权限、切换模式、查看工具调用和统计信息

## 说明

- Aite 关注的是 Claude Code 的日常使用体验，而不是替代 Claude Code CLI
- 如果你已经有 Claude Code 工作流，Aite 会更像一个更易用的桌面入口
